# claude-profile-cli 仕様書

複数の Claude Code アカウント(プロファイル)を切り替えて使うための CLI ツール。
エントリポイントは `claude-profile` のみ。

## 要件

- Node.js `>=18` (`package.json` の `engines`)

インストール手順は [README.md](./README.md) を参照。

## データ構造

```
~/.claude-profiles/
  <profile>/
    config/   # CLAUDE_CONFIG_DIR として使う独立した設定・履歴ディレクトリ
    token     # `claude setup-token` で発行した長期 OAuth トークン (chmod 600)
```

- ルートディレクトリは環境変数 `CLAUDE_PROFILES_DIR` で上書き可能。未設定時は `~/.claude-profiles`。
- プロファイルディレクトリ・親ディレクトリは `chmod 700`、トークンファイルは `chmod 600` で保護する。
- プロファイル名は英字または数字で始まり、英数字・`.`・`_`・`-` のみ使用できる。`..` や `/` などパストラバーサルにつながる名前は拒否する。
- 「プロファイルが存在する」とは `<profiles-dir>/<profile>/` ディレクトリが存在すること。「セットアップ済み」とはそれに加えて `token` ファイルが存在すること。

## プロファイル名の検証

`setup` / `remove` / `run` でプロファイル名を受け取るとき、次を満たさない場合は標準エラーに理由を表示して終了する。

- 英字または数字で始まる
- 英数字・`.`・`_`・`-` のみ使用する
- `..` や `/` などパストラバーサルにつながる名前は拒否する

エラーメッセージは次の情報を含む。

- 入力されたプロファイル名
- 拒否理由 (例: `` `..` は使えません。 ``)
- 有効な名前の例 (`work, my-profile`)
- 既存プロファイルがある場合はその一覧

## コマンド一覧

| コマンド | 役割 |
|---|---|
| `claude-profile setup <profile>` | 新規プロファイルを作成し、トークンを発行・保存する |
| `claude-profile list` | プロファイル一覧とセットアップ状態を表示する |
| `claude-profile remove [profile]` | プロファイルを削除する(省略時は対話選択) |
| `claude-profile run [profile] [claude args...]` | 指定プロファイルとして `claude` を起動する(省略時は対話選択) |

### ヘルプ・バージョン

```sh
claude-profile              # 全体ヘルプ(バージョン付き)
claude-profile --help
claude-profile -h
claude-profile --version
claude-profile -V
claude-profile help [command]   # コマンド別ヘルプ。省略時は全体ヘルプ
```

- 不明なサブコマンドを指定した場合はエラーメッセージを表示し、`claude-profile help` の実行を促して終了する。

### `claude-profile setup <profile>`

1. `<profiles-dir>/<profile>/config` を作成し、親ディレクトリ・プロファイルディレクトリを `chmod 700`。
2. 認証手順の案内メッセージを表示する。
3. `claude setup-token` を実行し、ブラウザでの OAuth 認証を求める。
4. 認証完了後、非表示プロンプト(入力内容を画面に表示しない)でトークン文字列の貼り付けを求める。
5. 貼り付けられたトークンを `<profiles-dir>/<profile>/token` に `chmod 600` で保存する。

- `<profile>` を省略、または `-` で始まる場合はコマンド別ヘルプを表示して終了する。
- プロファイル名が「プロファイル名の検証」に違反する場合はエラーで終了する。
- `claude setup-token` が失敗、またはトークンが空の場合は途中で中断する。

### `claude-profile list`

- プロファイルルート配下のディレクトリを列挙し、各プロファイルについて次をタブ区切りで表示する。
  - プロファイル名
  - セットアップ状態(`✓` = token あり / `✗ (token未設定)` = token なし)
  - config ディレクトリの絶対パス
- プロファイルが1つもない場合は `claude-profile setup <profile>` での作成を案内するメッセージのみ表示する。

### `claude-profile remove [profile]`

- `[profile]` を指定した場合はそのプロファイルを対象にする。省略した場合は既存プロファイル一覧から対話選択する(token 未設定のプロファイルも選択可)。
- プロファイル名が「プロファイル名の検証」に違反する場合はエラーで終了する。
- 指定/選択したプロファイルが存在しない場合はエラーで終了し、利用可能なプロファイル一覧を表示する。
- 削除前に確認プロンプト(デフォルト No)を表示する。
  - `No`(Enter のみ含む)の場合は `中断しました。` と表示して終了する。
  - 確認プロンプトをキャンセルした場合は終了する。
- `Yes` の場合、`<profiles-dir>/<profile>/` を再帰的に削除する。

### `claude-profile run [profile] [claude args...]`

- 第1引数が `-` で始まらない場合、それをプロファイル名として扱い、残りの引数を `claude` にそのまま渡す。
- プロファイル名が「プロファイル名の検証」に違反する場合はエラーで終了する。
- 第1引数が省略されている、または `-` で始まる場合、プロファイル名を対話選択で決定し、渡された引数はすべて `claude` への引数として扱う。
  - 対話選択の選択肢には token 未設定のプロファイルも表示されるが、`(token未設定)` と表示され選択不可(disabled)になる。
  - 選択可能なプロファイルが1つもない場合はエラーメッセージを表示して終了する。
  - 対話選択をキャンセルした場合は終了する。
- 決定したプロファイルの `token` ファイルが存在しない場合はエラーで終了し、`claude-profile setup <profile>` の実行を促す。
- トークンを読み込み、以下の環境変数を設定した上で `claude` を実行する。
  - `CLAUDE_CONFIG_DIR`: `<profiles-dir>/<profile>/config`
  - `CLAUDE_CODE_OAUTH_TOKEN`: トークンの内容
- `claude` プロセスの終了コードをそのまま自身の終了コードとする。

## セキュリティ・ロギング

- プロファイル名を検証し、パストラバーサルによるトークン・設定ディレクトリの読み書きを防ぐ。
- トークン本体はログに含めない。
- ユーザー向け出力(ヘルプ、一覧、対話メッセージ)は標準出力/標準エラー。診断ログは `LOGTAPE_LEVEL` 環境変数で詳細度を制御できる。
